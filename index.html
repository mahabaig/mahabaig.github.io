AZURE SOC LAB WALKTHROUGH

This project demonstrates hands-on experience in building a Microsoft Azure SOC environment, analyzing security events, and visualizing global attack data.

Step 1: Environment Setup

I created a free Microsoft Azure subscription and deployed a Windows 10 VM honeypot exposed to the internet via RDP (3389). To attract real attack traffic, I disabled certain protections like the firewall and Network Level Authentication.

Step 2: Data Collection & Monitoring

Connected the VM to a Log Analytics Workspace and integrated with Microsoft Sentinel to ingest SecurityEvent logs (e.g., Event ID 4625 for failed logons).

Step 4: Results & Insights

Within 24 hours, the VM recorded over 30,000 failed login attempts from multiple regions, revealing brute-force tactics like username cycling and predictable passwords.

Key Takeaways

  • Configured an end-to-end SIEM pipeline.
  • Built and tuned KQL queries for enrichment.
  • Gained threat visualization practice with Sentinel.

NETWORK TRAFFIC ANALYSIS & INTRUSION DETECTION LAB WALKTHROUGH

Captured, filtered, and analyzed packets with TCPDump and Wireshark to detect ARP scans and RDP activity patterns.

Step 1 — Initialize TCPDump

Executed tcpdump -i 1 -c 10 on Kali to capture live packets (ARP & TCP handshakes).

Step 2 — Protocol/Port Filters

Isolated RDP traffic (dst port 3389) and verified remote sessions.

Step 3 — Write & Read PCAP

Saved captures to .pcap (-w) and analyzed offline (-r) for forensic workflow.

Step 4 — Capture HTTP in Wireshark

Browsed the web while capturing; filtered with http.request.method == "CONNECT" and ip.src == 10.1.66.165.

Step 5 — Follow Streams

Used Follow → HTTP Stream to reconstruct requests/responses for inspection.

Step 6 — Detect ARP Scans

Filtered arp and ran discovery (nmap -sn) to see broadcast “who-has” patterns.

Key Takeaways

  • Captured and analyzed 50+ packets across ARP, TCP, HTTP.
  • Identified ARP scans and RDP anomalies in real time.
  • Practiced evidence collection with TCPDump & Wireshark.
  • Strengthened network forensics & IDS fundamentals.
